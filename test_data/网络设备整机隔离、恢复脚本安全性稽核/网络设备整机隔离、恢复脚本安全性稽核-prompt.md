# 网络设备整机隔离/恢复脚本安全性稽核skill描述

## 一、背景

针对网络设备版本升级、设备隔离更换等场景，场景共性是都要先对设备进行安全的整机隔离与重新入网操作，以安全的顺序关闭及恢复端口，避免对现网业务流量造成影响。

## 二、网络云资源池网络设备类型定义

| 设备类型           | 网络层级 | 设备名称关键字（基于此列信息判断设备类型与网络层级） |
| ------------------ | -------- | ---------------------------------------------------- |
| **业务TOR**        | 接入层   | S-TOR                                                |
| **管理TOR**        | 接入层   | M-TOR                                                |
| **存储TOR**        | 接入层   | ST-TOR                                               |
| **硬件管理TOR**    | 接入层   | DM-TOR                                               |
| **业务EOR**        | 核心层   | S-EOR                                                |
| **管理EOR**        | 核心层   | M-EOR                                                |
| **存储EOR**        | 核心层   | ST-EOR                                               |
| **DCGW**           | 核心层   | S-RT或E-RT                                           |
| **业务业支防火墙** | 核心层   | S-FW                                                 |
| **网管防火墙**     | 核心层   | M-FW                                                 |
| **负载均衡**       | 核心层   | LB                                                   |
| **CMNET防火墙**    | 出口层   | S-FW或C-FW                                           |
| **业务CE**         | 出口层   | NETCN                                                |
| **业支CE**         | 出口层   | NETBOSS                                              |
| **网管CE**         | 出口层   | NETNMS                                               |
| **CMNET CE**       | 出口层   | PC-CMNET或N-RT                                       |

***业务业支防火墙与CMNET防火墙存在类似的设备名称关键字S-FW，需要根据LLDP邻居信息进一步区分出设备类型。如邻居信息中发现对端设备名称包含PC-CMNET或N-RT，则属于CMNETF防火墙。***

## 三、功能描述

请基于上述背景，创建一个针对此类场景下整机隔离/恢复脚本的安全性稽核skill，具体如下（以DCGW为例）：

**被操作设备的LLDP数据**：各厂家（华为/中兴/华三/锐捷）通过数通CLI指令自行采集的原始LLDP数据。

**被操作设备的聚合端口与物理端口映射关系数据**：各厂家（华为/中兴/华三/锐捷）通过查询设备配置、聚合口状态等方式获取到的聚合端口与物理端口的映射关系，比如：Eth-Trunk1聚合端口有两个成员物理口，分别为40GE1/0/1和40GE1/0/2。

**被操作设备的整机隔离配置脚本**

**被操作设备的整机恢复配置脚本**

2、基于LLDP数据（如./NFV-D-HNZJI-02A-1801-AE02-S-RT-01-display lldp neighbor brief.txt，这是原始采集数据，需要做下解析）确认对端设备信息以及本端对应的物理端口信息。

***注：不同厂家、不同设备类型有不同的LLDP数据格式，需要对应做解析，后续再扩展。***

3、针对DCGW的LLDP信息，对上行、下行、互联物理端口进行分类，具体如下：

**上行端口**：对端设备为广电防火墙/CMNET防火墙/网管CE/业务CE/业支CE/业务业支防火墙/网管防火墙的本端物理端口。

**下行端口**：对端设备为业务EOR/负载均衡/管理EOR的本端物理端口。

**互联端口**：对端设备为DCGW的本端物理端口。

**剩余端口**：若经过上行、下行、互联端口筛选后仍存在部分本端物理端口，则单独列在报告中，供运维人员审核。

***注：其他设备类型也应该有类似的上、下、互联接口判断规则，可以先按设备类型创建占位，后续再补充。***

4、查看聚合端口与物理端口的映射关系数据（如./NFV-D-HNZJI-02A-1801-AE02-S-RT-01-display interface brief main.txt，这是原始采集数据，需要做下解析），主要针对性解决部分隔离/恢复脚本存在聚合端口的场景，聚合端口的上行/下行/互联属性，与其成员物理口保持一致。

| 设备名称                          | 物理端口             | 所属聚合端口 |
| --------------------------------- | -------------------- | ------------ |
| NFV-D-HNZJI-02A-1801-AE02-S-RT-01 | GigabitEthernet4/0/5 | Eth-Trunk2   |
| NFV-D-HNZJI-02A-1801-AE02-S-RT-01 | 50\|100GE3/1/0       | Eth-Trunk1   |

***注：不同厂家、不同设备类型有不同的聚合端口-物理端口数据格式，需要对应做解析，后续再扩展。***

5、DCGW端口隔离/恢复顺序定义：

**端口隔离顺序**：上行->下行->互联。

**端口恢复顺序**：互联->上行->下行。

***注：其他设备类型也应该有类似的端口隔离/恢复顺序，可以先按设备类型创建占位，后续再补充。***

6、读取DCGW的整机隔离操作脚本（./NFV-D-HNZJI-02A-1801-AE02-S-RT-01-隔离脚本.txt）和整机恢复操作脚本（./NFV-D-HNZJI-02A-1801-AE02-S-RT-01-恢复脚本.txt），根据步骤4中的端口隔离/恢复顺序，对操作脚本进行稽核，包括：

（1）对隔离操作脚本的端口关闭顺序、端口关闭完整性、准确性进行自动稽核，避免端口关闭顺序错误、端口少关/多关。

（2）端口恢复（设备升级或更换后重新入网）步骤同理。

7、基于稽核结果，输出稽核报告，除了分析结论外，报告必须包含如下信息，作为证据：

（1）列出LLDP的原始信息，如能关联到聚合口，新增一列体现聚合口名称，再新增一列标注其属于上行/下行/互联/剩余端口。

（2）在原有端口隔离/恢复脚本上，逐个端口标注其配置的合法性，必须有依据地进行分析与判断。
