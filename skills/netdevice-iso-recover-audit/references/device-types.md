# 网络云资源池网络设备类型定义

本文档定义网络云资源池中涉及的设备类型、网络层级与设备名称关键字的对应关系。稽核脚本在识别被操作设备时，**优先以设备本端系统名（sysname）作为输入**。当被稽核文件命名清晰包含设备名时，直接使用；当仅靠文件名无法判断时，可结合配置首行 `sysname` 进一步确认。

## 设备类型清单

| 设备类型           | 网络层级 | 设备名称关键字（用于识别） |
| ------------------ | -------- | -------------------------- |
| 业务 TOR           | 接入层   | `S-TOR`                    |
| 管理 TOR           | 接入层   | `M-TOR`                    |
| 存储 TOR           | 接入层   | `ST-TOR`                   |
| 硬件管理 TOR       | 接入层   | `DM-TOR`                   |
| 业务 EOR           | 核心层   | `S-EOR`                    |
| 管理 EOR           | 核心层   | `M-EOR`                    |
| 存储 EOR           | 核心层   | `ST-EOR`                   |
| DCGW               | 核心层   | `S-RT` 或 `E-RT`           |
| 业务业支防火墙     | 核心层   | `S-FW`                     |
| 网管防火墙         | 核心层   | `M-FW`                     |
| 负载均衡           | 核心层   | `LB`                       |
| CMNET 防火墙       | 出口层   | `S-FW` 或 `C-FW`           |
| 业务 CE            | 出口层   | `NETCN`                    |
| 业支 CE            | 出口层   | `NETBOSS`                  |
| 网管 CE            | 出口层   | `NETNMS`                   |
| CMNET CE           | 出口层   | `PC-CMNET` 或 `N-RT`       |
| 广电防火墙         | 出口层   | `CBN`                      |

## 名称关键字冲突处理（重要）

`S-FW` 同时被「业务业支防火墙」和「CMNET 防火墙」使用，必须结合 LLDP 邻居信息进一步区分：

- 如果对端设备名称中包含 `PC-CMNET` 或 `N-RT`，则本端属于 **CMNET 防火墙**。
- 否则按业务业支防火墙处理。

> 其他关键字目前未发现冲突；若后续发现新冲突，应在本文件追加冲突说明。
